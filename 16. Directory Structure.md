## Directory Structure
  - ### 1️⃣ Flat 디렉토리 구조 <br><br> <img src="https://user-images.githubusercontent.com/35948339/137866596-28a8ddf7-88dd-4dc9-9d3f-1c56d53cb0d8.png" width=500>
    - #### File System내에 하나의 디렉토리만 존재 (Single-level 디렉토리 구조)
    - #### ex. 초창기 MP3 (폴더 구분 없이 mp3 파일만 넣었을 때)
    - #### ❌ 문제점
      - #### 파일 네이밍 문제 <br><br> → 폴더가 하나면, 이름이 겹칠 가능성이 높다.
      - #### 파일 관리 문제 <br><br> → 다중 사용자 환경이라면, 문제가 더욱 커진다.

  - ### 2️⃣ 2-Level 디렉토리 구조 <br><br> <img src="https://user-images.githubusercontent.com/35948339/137866868-74446695-5397-4837-83e6-8da828cc9226.png" width=500>
    - #### 사용자 마다 하나의 디렉토리 배정
    - #### 루트 폴더에 단 하나의 자식 폴더
    - #### ❌ 문제점
      - #### Sub-directory 생성 불가능 <br><br> 1️⃣ 과 같은 불편함 남아있음
      - #### 사용자 간 파일 공유 문제 <br><br> → 파일 하나를 공유하기 위해서는 폴더 전체를 공유해야 하므로 보안상 문제 발생
